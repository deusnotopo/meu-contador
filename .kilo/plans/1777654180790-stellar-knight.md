## Riscos

- **Risco 1:** Refatoração SQL pode introduzir bugs se não for cuidadosa
  - Mitigação: O código atual é seguro, então a refatoração é apenas melhoria de boas práticas
  - Testar exaustivamente com dados reais
  
- **Risco 2:** Mudanças na API podem afetar outras partes do sistema
  - Mitigação: Verificar todas as chamadas antes

- **Risco 3:** Correção de etapa/status pode alterar comportamento do CRM
  - Mitigação: Manter compatibilidade reversa, preservar status customizados
