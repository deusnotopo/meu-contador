## 🎯 Regras de Ouro para Antigravity

### Frontend (React + TypeScript)

#### ✅ SEMPRE:
1. **Usar TypeScript estrito**
   ```typescript
   // ❌ ERRADO
   const value: any = getData();
   
   // ✅ CERTO
   const value: User = getData();
   ```

2. **Definir Props explicitamente**
   ```typescript
   // ❌ ERRADO
   function Component(props) {
     return <div>{props.name}</div>;
   }
   
   // ✅ CERTO
   interface Props {
     name: string;
     age?: number;
   }
   function Component({ name, age }: Props) {
     return <div>{name}</div>;
   }
   ```

3. **Usar hooks customizados para lógica complexa**
   ```typescript
   // ❌ ERRADO - lógica no componente
   function Transactions() {
     const [data, setData] = useState([]);
     useEffect(() => {
       fetch('/api/transactions')
         .then(res => res.json())
         .then(setData);
     }, []);
   }
   
   // ✅ CERTO - hook customizado
   function useTransactions() {
     const [data, setData] = useState<Transaction[]>([]);
     const [loading, setLoading] = useState(true);
     const [error, setError] = useState<Error | null>(null);
     
     useEffect(() => {
       fetchTransactions()
         .then(setData)
         .catch(setError)
         .finally(() => setLoading(false));
     }, []);
     
     return { data, loading, error };
   }
   ```

4. **Tratar erros com Error Boundaries**
   ```typescript
   // ✅ SEMPRE envolver componentes complexos
   <ErrorBoundary featureName="Dashboard">
     <Dashboard />
   </ErrorBoundary>
   ```

5. **Usar Suspense para lazy loading**
   ```typescript
   const LazyComponent = lazy(() => import('./Component'));
   
   <Suspense fallback={<LoadingSkeleton />}>
     <LazyComponent />
   </Suspense>
   ```

#### ❌ NUNCA:
1. Não usar `any` - sempre definir tipo correto
2. Não mutar estado diretamente
3. Não usar índices como key em listas
4. Não fazer fetch em useEffect sem cleanup
5. Não ignorar erros de TypeScript

### Backend (Fastify + Prisma)

#### ✅ SEMPRE:
1. **Validar entrada com Zod**
   ```typescript
   import { z } from 'zod';
   
   const transactionSchema = z.object({
     amount: z.number().positive(),
     description: z.string().min(1).max(255),
     date: z.string().datetime(),
   });
   
   // Na rota
   fastify.post('/transactions', {
     schema: {
       body: transactionSchema,
     },
   }, async (request, reply) => {
     // request.body já está validado e tipado
   });
   ```

2. **Usar tipos do Prisma**
   ```typescript
   import { Prisma } from '@prisma/client';
   
   // ✅ Usar tipos gerados
   type User = Prisma.UserGetPayload<{}>;
   type TransactionWithCategory = Prisma.TransactionGetPayload<{
     include: { category: true };
   }>;
   ```

3. **Tratar erros de forma consistente**
   ```typescript
   // ✅ Wrapper de erro consistente
   try {
     const result = await prisma.transaction.create({ data });
     return { success: true, data: result };
   } catch (error) {
     if (error instanceof Prisma.PrismaClientKnownRequestError) {
       throw new AppError('Database error', 400, error.code);
     }
     throw error;
   }
   ```

4. **Usar middleware para autenticação**
   ```typescript
   fastify.addHook('preHandler', async (request, reply) => {
     if (request.routeOptions.url?.startsWith('/api/protected')) {
       await request.jwtVerify();
     }
   });
   ```

#### ❌ NUNCA:
1. Não concatenar strings em queries SQL
2. Não expor erros internos ao cliente
3. Não logar dados sensíveis
4. Não usar `as` para type assertion sem necessidade

## 🚨 Erros Comuns e Como Evitá-los

### 1. Quebra de Imports
**Problema**: Mover/renomear arquivos sem atualizar imports
**Solução**: 
- Usar IDE para refatorar
- Buscar por referências antes de mover
- Testar build após mudanças

### 2. Tipos Incompatíveis
**Problema**: Frontend espera um tipo, backend envia outro
**Solução**:
- Compartilhar tipos entre frontend/backend
- Usar Zod para validar em runtime
- Testar integração

### 3. Memory Leaks
**Problema**: useEffect sem cleanup
**Solução**:
- Sempre retornar cleanup function
- Usar AbortController para fetch
- Cancelar subscriptions

### 4. Race Conditions
**Problema**: Múltiplas atualizações de estado
**Solução**:
- Usar useCallback para funções
- Implementar debounce
- Usar AbortController

### 5. Erros Não Tratados
**Problema**: Promises sem catch
**Solução**:
- Try/catch em async/await
- .catch() em promises
- Error Boundaries
